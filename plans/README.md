## Findings considered and rejected (do not re-audit)

- **Committed secrets in `.env.local`** — FALSE POSITIVE as a repo issue. The
  file is gitignored, untracked (`git ls-files --error-unmatch` errors), and
  absent from history. (The auditing subagent read the on-disk values; those
  were NOT propagated here. Operator was advised out-of-band to rotate the
  on-disk dev credentials if that file was ever shared.)
- **`await db.delete(obj)` "raises AttributeError on AsyncSession"**
  (`correlation.py:340,387`) — FALSE. `AsyncSession.delete()` is a valid
  awaitable coroutine in SQLAlchemy 2.0; the code is correct idiom.
- **Settings monolith decomposition** — already done. `SettingsHub` +
  `pages/settings/SettingsSection` exist; the old 2477-line `Settings.tsx`
  pattern was already broken up.
- **CORS misconfiguration** (`main.py:222`) — by-design. Origins are hardcoded
  dev localhost values (not wildcard) with credentials; prod is same-origin.
  Minor at most.
- **Missing CSRF protection** — N/A. Auth is JWT (Authorization header), and a
  CSRF middleware (`core/csrf.py`) already exists for cookie surfaces.
- **JWT algorithm confusion** — N/A. `jwt.decode(..., algorithms=[...])` is
  pinned (`core/security.py`).
- **Permanent `User.totp_secret` plaintext** — FALSE. It is encrypted at rest
  via a property (`user.py:65-78`). Only the transient setup token was plaintext
  → that is the (narrower) basis for Plan 004.
- **Alert dedup minute-bucket collision** (`alerts.py:48`) — by-design dedup
  window; not a bug.
- **`get_nested_value` unbounded recursion / stack overflow** — speculative; the
  function is iterative, not recursive. Rejected.
- **Frontend list virtualization** — low current impact (pages cap at ~100 rows
  with pagination); not worth the complexity now.
